# Delivery Folder

GitHub workflow, CI/CD pipelines, release management.

## Doc Index
- `github/workflows-index.md` — Map of every GH Actions workflow + purpose
- `github/branch-strategy.md` — Branch naming, protection rules
- `github/pr-template.md` — Mirror of `.github/pull_request_template.md`
- `github/issue-templates/` — Mirror of `.github/ISSUE_TEMPLATE/`
- `github/codeowners.md` — Annotated CODEOWNERS with rationale
- `github/labels.md` — Standard label taxonomy
- `pipelines/ci.md` — What CI runs, when, gates
- `pipelines/cd.md` — Deploy stages: dev → staging → prod
- `pipelines/release-process.md` — Versioning, changelog, rollback
- `pipelines/environments.md` — URLs, owners, data sensitivity
- `runbooks/deploy.md`, `rollback.md`, `hotfix.md` — Operational runbooks
- `releases/` — Release notes archive (generated by `content-documentation` agent)

## Instructions for Claude
- Use `mcp__MCP_DOCKER__create_pull_request` to open PRs.
- Use `mcp__MCP_DOCKER__list_issues` and `mcp__MCP_DOCKER__add_issue_comment` for issue triage.
- Never push directly to `main` — branch protection requires PR + review + green CI.
- Release readiness: invoke `.claude/skills/release-readiness-check.md` before tagging.
