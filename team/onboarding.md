# Onboarding

Welcome to the Product Office. This repo is the team's operating system — read it like a codebase.

## Day 1
- [ ] Add yourself to `team/directory.md` via a PR
- [ ] Read `CLAUDE.md` (the map) and `governance/framework.md` (the constitution)
- [ ] Install VS Code Insiders + open `Product-Office.code-workspace`
- [ ] Install the recommended extensions when prompted
- [ ] Install Claude Code if you haven't, and verify it can read this repo

## Day 2
- [ ] Run `/pm-status` — get a feel for what's in flight
- [ ] Skim the 11 agents in `agents/`
- [ ] Skim the 7 Nine-Organ skills (`/office-hours`, `/plan-ceo-review`, etc.)
- [ ] Read the PRD style guide: `.claude/skills/writing-guide.md`

## Week 1
- [ ] Shadow a PRD review
- [ ] Run `/new-prd` on a test feature, open a PR, get it reviewed
- [ ] Read the most recent 3 ADRs in `product/decisions/`
- [ ] Skim `quality/definition-of-done.md` and `quality/test-strategy.md`

## Ongoing
- All work happens on branches → PR → review → merge
- Use slash commands instead of doing things manually when possible
- When in doubt, route to the right agent (see `CLAUDE.md` registry)
