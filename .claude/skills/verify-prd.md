# Verify PRD

> Run before opening or merging a PRD PR. The CI workflow `validate-docs.yml` enforces the same checks — failing locally first saves a round-trip.

## Checklist
For the PRD at `product/prds/active/<feature>/`:

- [ ] **Frontmatter complete** in `PRD-Summary.md`: `name`, `status`, `owner`, `stage`
- [ ] **Lean one-pager** (`01-lean-onepager.md`) exists and is ≤1 printed page
- [ ] **If Stage 2:** Shape Up pitch (`02-shape-up-pitch.md`) exists with appetite + breadboard + rabbit holes + no-gos
- [ ] **Metric link:** PRD frontmatter `links.metrics` points to a real anchor in `analytics/metrics.md`
- [ ] **Acceptance criteria link:** `links.acceptance_criteria` points to an existing `quality/acceptance-criteria/<feature>.md` with at least one Gherkin scenario per requirement
- [ ] **SLO link** (if user-facing): `links.slo` points to a real entry in `observability/slos.md`. Internal-only PRDs can mark `slo: N/A — internal`
- [ ] **Stakeholders / RACI:** at least one R, one A, one C, one I in the RACI table (or generated by `stakeholder-intelligence` agent)
- [ ] **Non-goals listed:** explicit "what this is not"
- [ ] **Risks listed:** at least 3
- [ ] **Tracking issue / PR** linked
- [ ] **Banned phrases removed** (per `writing-guide.md`)

## How to Run Locally
```bash
python scripts/validate-prd-links.py product/prds/active/<feature>/
```
Exit code 0 = pass. Non-zero = fix what it reports, then re-run.

## On Failure
Fix the missing link rather than disabling the check. If a metric genuinely doesn't exist yet, add a stub to `analytics/metrics.md` with `status: proposed`.
